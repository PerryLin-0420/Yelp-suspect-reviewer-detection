# Yelp 可疑用戶偵測

## 概述

針對 Yelp Open Dataset（1,987,841 名用戶）的無監督異常偵測，不依賴標記資料。  
從兩個完全獨立的維度出發，各自找到行為異常的帳號，再透過店家群聚分析確認協調行為。

---

## 分析架構

```
[資料庫]
    │
    ├─► 分支一：時段 K-Means          分支二：帳號生命週期 K-Means
    │   用戶 × 24小時發評指紋               用戶 × 50格生命週期密度向量
    │   ↓                                   ↓
    │   異常群（C10、C19）→ Sub3/Sub4        單評論帳號 / 爆發型帳號
    │   ↓                                   ↓
    │   餘弦相似度 → 4,037 候選人            店家群聚驗證
    │   ↓
    │   店家群聚驗證（K1 Tier-2）
    │
    └─► 最終除名清單（3,439 人）
    │
    └─► 回頭驗證（verify/）
```

## 分支一：時段 K-Means

### 設計理念

真實用戶的發評時段由作息與時區塑造，不會無緣無故集中在清晨。  
**Yelp 以用戶本地時間儲存時間戳**（實證：PA 用戶發評高峰 = 18:00 美東時間），因此早晨異常代表本地時間的真實偏離。

### 特徵建構

每位用戶 → **24 維概率向量**（小時發評比例）

- 按年歸一化後加總，再次歸一化，消除年份差異
- ≥5 則評論的用戶才進入分析，共 **287,000 人**

### 聚類與可疑群識別

MiniBatchKMeans（K=20），以**熵**和**早晨比例（06:00–11:59）**評估各群。

| 可疑群 | 早晨比例 | 特徵 |
|--------|---------|------|
| Cluster 10 | 顯著偏高 | 正常清醒時段接近零 |
| Cluster 19 | 顯著偏高 | 同上 |

**交叉驗證**：計算各子群用戶所評店家的營業時間熱力圖

- C10-Sub3（n=420）：早晨評論中只有 **9%** 落在店家已開業時段（正常群：43%）
- C10-Sub4（n=357）：早晨評論中只有 **14%** 落在店家已開業時段

### 全體評分

以 Sub3+Sub4 建立異常中心點，對全體 287k 用戶算餘弦相似度。  
門檻 = 已知異常用戶分數的第 5 百分位數（**0.546**），得到 **4,037 名候選人**（1.4%）。

### K1 Tier-2 衍生驗證

4,037 名 K1 候選人中，只有 **9 人**與 Lifetime 除名清單重疊——兩條路線幾乎正交。  
剩餘 **4,028 名純 K1 用戶**，以相同的店家群聚邏輯再驗證：

- 45 家店家出現協調刷評（span < 7d）→ **78 人**確認，納入除名清單
- 沒有任何店家達到 suspicion_score ≥ 0.5
- 對應的 closed_pct 分析：mean = 0.847（正常用戶 0.721），確認早晨異常為真實偏離，不是時區干擾

## 分支二：帳號生命週期 K-Means

### 設計理念

無論時區，可疑帳號的行為會集中在帳號生命的某個時間點：

- **單評論**：創號後只發一則評論，消失
- **爆發後消失**：創號後短時間密集發評，之後沉默

### 全局歸一化

```
position = (評論日期 − 帳號創立日期) / 全局最大間距（≈ 6,241 天）
```

所有用戶映射到共同的 [0, 1] 軸，消除加入時間不同的影響。

### 單評論用戶（n = 1,135,945）

一則評論的用戶以 50 格 one-hot 向量聚類。

**關鍵發現**：Cluster 1（n=320,060），mean position ≈ 0.003  
→ 28% 的單評論用戶在帳號創立後 19 天內發出唯一評論，之後消失。

### 多評論用戶（n = 851,896）

50 格概率向量，MiniBatchKMeans（K=20），以熵評估。

| 群 | 熵 | 峰值位置 | 意義 |
|----|----|---------|------|
| Cluster 1（n=91,597） | 0.31 | 0.00 | 最典型的爆發後消失 |

**爆發群定義**：`peak_position ≤ 0.10` 且 `entropy < 2.0`  
→ Cluster 1、5、7、9、14，共 217,641 人

## 店家群聚與時間集中度

### 核心邏輯

高覆蓋率 + 短時間窗口 = 協調刷評  
高覆蓋率但分散多年 = 自然累積（非信號）

### 指標

| 指標 | 定義 |
|------|------|
| `coverage_rate` | 可疑用戶數 / 該店家總評論數 |
| `span_days` | 最早至最晚一筆可疑評論的間距（天） |
| `dense_30d_pct` | 可疑評論中落在最密集 30 天窗口的比例 |
| `suspicion_score` | `coverage_rate × dense_30d_pct` |

**協調確認門檻**：`span_days < 7`  
**高重疊門檻**：`suspicion_score ≥ 0.5`（覆蓋率與時間集中度同時 ≥ 50%）

> 資料集總用戶：**1,987,841 人**  
> 輸出：`lifetime kmeans/result/final/removal_final.parquet`

| 類別 | 人數 | 佔比 | 判定條件 |
|------|------|------|---------|
| 協調刷評（單評論） | 1,424 | 0.072% | ≥ 2 個可疑帳號評同一店家，span < 7d |
| 協調刷評（爆發型） | 744 | 0.037% | 同上 |

| **合計** | **3,439** | **0.173%** | |

## 回頭驗證：方法論獨立性

`verify/scripts/verify_all.py` 將 3,439 名 flagged 用戶放回所有分析維度。

| 指標 | 數值 |
|------|------|
| 出現在 K1 空間的 flagged 用戶 | 514 / 3,439（15%） |
| 這 514 人的平均餘弦相似度 | 0.168（K1 門檻 = 0.546） |
| 在 Lifetime 單評論系統 | 2,617 人（76%） |
| closed_pct：正常用戶 mean | 0.721 |
| closed_pct：flagged mean | 0.740（Δ = +0.018） |

**核心發現**：flagged 用戶在「不是因此被抓」的維度上，與正常用戶幾乎無異。這不是循環論證的問題——恰恰相反，**正交性證明每條路線偵測的是真正不同的底層行為**：

| 類別 | 行為模式 | 推斷動機 |
|------|---------|---------|
| `single_coordinated` | 多帳號 7 天內評同一店，消失 | 受僱一次性刷評 |
| `burst_coordinated` | 帳號早期爆發後沉默，共享目標店家 | 批量刷評或收購帳號 |
| `single_high_overlap` | 所評店家被可疑帳號高度覆蓋 | 刷評生態圈的參與者 |
| `k1_coordinated` | 系統性早晨發評 + 共享目標店家 | 自動化腳本或遠端操控 |

## 門檻選擇依據

| 門檻 | 選擇理由 |
|------|---------|
| `MIN_SUSPECT_USERS = 2` | 兩個一次性帳號在 7 天內獨立評同一家小店，機率本就極低；要求 3 個會錯失大量真實協調行為 |
| `span_days < 7` | 3 天內 = 近乎確定的協調；7 天仍難以被自然流量解釋；14 天後口碑傳播成為合理替代解釋 |
| `suspicion_score ≥ 0.5` | 覆蓋率與時間集中度同時過半，單一條件不構成信號 |
| 爆發型帳號不直接除名 | 217,641 人的生命週期模式異常，但高活躍真實用戶特徵相同；只有 744 人有直接協調證據才納入 |

## 偵測信號總覽

| 信號 | 分支 | 強度 |
|------|------|------|
| 早晨發評比例異常（本地時間） | 時段 K-Means | 強；已以店家營業時間交叉驗證 |
| 評論時間 vs 店家營業時間不符 | 時段 K-Means | 時區無關，獨立佐證 |
| 餘弦相似度（全體 287k） | 時段 K-Means | 將異常推廣至全體用戶 |
| K1 候選人 + 店家協調 | 時段 K-Means × 店家群聚 | 行為佐證確認 |
| 單評論帳號 | 生命週期 K-Means | 廣泛；單獨精確度有限 |
| 爆發後消失模式 | 生命週期 K-Means | 中等；加上店家群聚後增強 |
| 協調針對（span < 7d） | 店家群聚 | 精確度最高；人數最少 |
